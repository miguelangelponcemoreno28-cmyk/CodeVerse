Cómo Crear tus Propios Snippets en VS Code

Aprende a crear fragmentos de código reutilizables para acelerar tu desarrollo

15 minutos Principiante VS Code

Un snippet es un fragmento de código que puedes insertar rápidamente escribiendo un atajo y presionando Tab. En este tutorial aprenderás a crear snippets personalizados para acelerar tu flujo de desarrollo.

Requisito:

Tener VS Code instalado. Los snippets funcionan igual en Windows, macOS y Linux.

¿Por qué usar snippets?

Velocidad

Ahorra tiempo escribiendo código repetitivo

Consistencia

Garantiza que sigas siempre el mismo patrón

Menos errores

Reduce errores tipográficos

Profesionalismo

Mantén estándares en tu equipo

1 Abrir el Editor de Snippets

En Windows/Linux:

Presiona Ctrl + Shift + P

En Mac:

Presiona Cmd + Shift + P

En la paleta de comandos:

Escribe: Configure User Snippets

Alternativa:

También puedes ir a Archivo → Preferencias → Fragmentos de usuario

2 Entender la Estructura JSON

Los snippets se escriben en formato JSON. Aquí está la estructura básica:

{
  "Nombre del Snippet": {
    "prefix": "atajo",
    "body": [
      "línea 1",
      "línea 2",
      "línea 3"
    ],
    "description": "Descripción corta"
  }
}

Explicación de campos:

prefix: El atajo que escribes para activar el snippet (ej: "htmlbase")

body: Array con las líneas de código. Cada línea es un string

description: Descripción que aparece en el autocompletado

3 Crear tu Primer Snippet

Vamos a crear un snippet simple. Copia este código en tu archivo de snippets:

{
  "React Functional Component": {
    "prefix": "rfc",
    "body": [
      "import React from 'react';",
      "",
      "export default function ${1:ComponentName}() {",
      "  return (",
      "    <div>",
      "      $0",
      "    </div>",
      "  );",
      "}"
    ],
    "description": "Componente funcional de React"
  }
}

Ahora prueba: abre un archivo JavaScript/React, escribe rfc y presiona Tab

Resultado:

Se insertará automáticamente el código del componente y el cursor estará en ComponentName para que edites.

4 Variables y Placeholders

Los snippets usan placeholders para posiciones del cursor. Los más importantes son:

$1, $2, $3 - Posiciones de tabulación

Cada número es una posición donde el cursor saltará cuando presiones Tab:

{
  "Función con argumentos": {
    "prefix": "fn",
    "body": [
      "function ${1:nombreFuncion}(${2:args}) {",
      "  ${3:// tu código aquí}",
      "  return ${0};",
      "}"
    ],
    "description": "Función de JavaScript"
  }
}

${1:valor_por_defecto} - Valores predeterminados

Puedes poner un valor que aparezca automáticamente:

{
  "Console log": {
    "prefix": "cl",
    "body": [
      "console.log(${1:'mensaje'});"
    ],
    "description": "Imprime en consola"
  }
}

5 Ejemplos Prácticos Listos para Usar

1. Snippet de Clase Python 

{
  "Python Class": {
    "prefix": "pyclass",
    "body": [
      "class ${1:ClassName}:",
      "    def __init__(self, ${2:params}):",
      "        ${3:# Tu código}",
      "        pass",
      "",
      "    def ${4:method_name}(self):",
      "        ${0:pass}"
    ],
    "description": "Clase de Python"
  }
}

2. Snippet de Try-Catch JavaScript 

{
  "Try Catch": {
    "prefix": "tc",
    "body": [
      "try {",
      "  $1",
      "} catch (error) {",
      "  console.error('Error:', error);",
      "  $0",
      "}"
    ],
    "description": "Bloque try-catch"
  }
}

3. Snippet de Async/Await 

{
  "Async Function": {
    "prefix": "async",
    "body": [
      "async function ${1:nombreFuncion}() {",
      "  try {",
      "    const resultado = await ${2:promesa};",
      "    ${3:// procesa resultado}",
      "    return resultado;",
      "  } catch (error) {",
      "    console.error(error);",
      "  }",
      "}"
    ],
    "description": "Función asincrónica"
  }
}

Consejos Pro

Prefixes cortos:

Usa atajos de 2-4 caracteres (rfc, fc, tc) para máxima velocidad

Un snippet = Una idea:

Mantén snippets pequeños y específicos, no hagas "mega-snippets"

Nombres descriptivos:

En el campo "description" sé claro sobre qué hace

Prueba antes de compartir:

Asegúrate de que el snippet funciona antes de usarlo en el equipo

Errores Comunes y Soluciones

Error: "Invalid JSON"

Asegúrate de poner comas entre propiedades y comillas correctas

El snippet no aparece:

Recarga VS Code o cierra y abre el archivo

Las variables no funcionan:

Verifica que uses $1, $2, $0 (con $ adelante)

Compartir Snippets con tu Equipo

Los archivos de snippets se guardan en:

Windows:
%APPDATA%\Code\User\snippets
Mac:
~/Library/Application Support/Code/User/snippets
Linux:
~/.config/Code/User/snippets

Puedes:

  1. 1 Compartir archivos .code-snippets por email
  2. 2 Subirlos a un repositorio Git
  3. 3 Usar extensiones de VS Code que incluyen snippets

Recursos Adicionales

  • Documentación oficial: code.visualstudio.com/docs/editor/userdefinedsnippets
  • Variables especiales: Puedes usar $TM_FILENAME, $TM_LINE_NUMBER, etc.
  • Expresiones regulares: Los snippets avanzados pueden incluir transformaciones
